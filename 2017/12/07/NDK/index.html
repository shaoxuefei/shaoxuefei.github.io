JNI

简介

  • 定义：Java Native Interface,即Java本地接口

  • 作用：使用Java与本地其他类语言(如c、c++)交互

    即在Java代码里调用c、c++等语言的代码或c、c++代码调用Java代码

  • 特别注意：

    • JNI是调用的Native语言的一种特性
    • JNI是属于Java，与Android无直接关系，需要通过NDK来与Android建立关系

      为什么要有JNI

  • 背景：实际使用中，Java 需要与 本地代码 进行交互

  • 问题：因为 Java 具备跨平台的特点（java的跨平台：Java可以定义其他语言的调用方法,也就是向外抛出调用的对接方法，也就是其他语言可以调用Java本地定义的方法），所以Java 与 本地代码交互的能力非常弱
  • 解决方案： 采用 JNI特性 增强 Java 与 本地代码交互的能力

实现步骤

  • 在Java中声明Native方法（即需要调用的本地方法）
  • 编译上述 Java源文件javac（得到 .class文件）
  • 通过 javah 命令导出JNI的头文件（.h文件）

  • 使用 Java需要交互的本地代码 实现在 Java中声明的Native方法 

    如 Java 需要与 C++ 交互，那么就用C++实现 Java的Native方法
  • 编译.so库文件
  • 通过Java命令执行 Java程序，最终实现Java调用本地代码

NDK

NDK简介

  • 定义：Native Development Kit，是 Android的一个工具开发包 

    NDK是属于 Android 的，与Java并无直接关系
  • 作用：快速开发C、 C++的动态库，并自动将so和应用一起打包成 APK
    即可通过 NDK在 Android中 使用 JNI与本地代码（如C、C++）交互

  • 应用场景：在Android的场景下 使用JNI

    即 Android开发的功能需要本地代码（C/C++）实现
  • 特点
    • 性能方面
      • 运行效率高 : 一般需要高性能的计算算法工作，会用c/c++，机器语言执行效率高(生成.so文件)
      • 代码安全性高 : java是办解释型语言，容易被反编译后得到源码，而使用本地的代码(如c、c++)则不会，可以提高系统的安全性
    • 功能方面
      • 功能拓展性好 ： 除了使用java的开源库，还可以使用c\c++的开源库
    • 使用方面
      • 易于代码复用和移植 ： 用c\c++开发的代码不仅可以在Android中使用，还可以在其他类型平台使用
    • 额外注意
      • NDK提供了把.so和.apk打包的工具–而JNI开发并没有该功能，其只是生成.so文件，或者说是c\c++调用本地java方法依靠是JNI的原因，依靠JNI内部定义的本地java方法接口
      • NDK 提供的库有限，仅用于处理算法效率和敏感的问题
      • 提供交叉编译，用于生成特定的cpu平台的动态库：也可已理解说是因为在androidstudio中jniLib中声明的支持cpu的型号中的.so文件并不是都是一样的，不可以直接创建文件夹然后将别的文件夹中.so文件赋值过去就可以了(此处应该可以看出jniLibs–命名的含义也就是看似JNI生成的.so的文件的集合命名的，其实JNI只是定义了跟c\c++交互的接口规则，最终是NDK将其打包成.so并和应用一起打包成apk)

使用步骤

  • 配置 Android NDK环境
  • 创建 Android 项目，并与 NDK进行关联
  • 在 Android 项目中声明所需要调用的 Native方法

  • 使用 Android需要交互的本地代码 实现在Android中声明的Native方法 

    比如 Android 需要与 C++ 交互，那么就用C++ 实现 Java的Native方法
  • 通过 ndk - bulid 命令编译产生.so库文件
  • 编译 Android Studio 工程，从而实现 Android 调用本地代码

Android CPU

  • 获取手机的cpu架构信息

    1、adb shell  
2、cat /proc/cpuinfo

  • 型号
    Android系统目前支持以下七种不同的CPU架构：ARMv5，ARMv7 (从2010年起)，x86 (从2011年起)，MIPS (从2012年起)，ARMv8，MIPS64和x86_64 (从2014年起)，每一种都关联着一个相应的ABI-简称：应用程序二进制接口（Application Binary Interface）定义了二进制文件（尤其是.so文件）如何运行在相应的系统平台上，从使用的指令集，内存对齐到可用的系统函数库。在Android系统上，每一个CPU架构对应一个ABI：armeabi，armeabi-v7a，x86，mips，arm64-v8a，mips64，x86_64。

  • 兼容
    很多设备都支持多余一种的ABI：7中cpu的设备统一都可以运行含有arembi的二进制包(也就是apk文件),ARM64和x86设备也可以同时运行armeabi-v7a和armeabi的二进制包,但是最好是针对不同的平台，app中设置不同的.so文件，这样可以提高性能，比如你手机是x86_64,但是app中的.so只用x86的那么，相当于64手机系统运行32位的ART和Android组件，将丢失64的性能(webview,media,art)
