Fragment

发表于 2017-10-11

Fragment加载方式

FragmentManager fragmentManager=getFragmentManager();
FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();//开启Fragment的事务
//fragmentTransaction.add(R.id.mFrameLayout,homePageFragment);
fragmentTransaction.replace(R.id.mFrameLayout,homePageFragment,"Home");//启动时设置Tag
fragmentTransaction.commit();
  • replace()是先执行move操作然后在add(),当在一个FragmentA上启动另一个FragmentB时，运用replace()会直接将上一个FragmentA移除掉，然后再add-FragmentB；
  • add()是直接添加Fragment呈现，当同一FragmentLayout上边由FragmentA启动到FragmentB时，FragmentB会把底部的FragmentA呈现出来，相当于两个重叠的Fragment,第一种解决方法就是直接在FragmentB的布局中设置背景颜色；第二种就是在启动FragmentB时把FragmentA隐藏，这时如果获取FragmentA的对象不方便的话，可以通过getFragmentManager().findFragmentByTag("Home")//通过Tag来获取对应的Fragment或者是通过设置Fragment的Id：fragmentManager.findFragmentById("xx"),这时如果遇见非叠加时的hide是需要在显示时show出来；

    后退栈

    getFragmentManager().beginTransaction().addToBackStack(String name)
    是将对应的Fragment加入后退栈中，一般在事务提交commit()之前执行,
    fragmentTransaction.addToBackStack(null);
    fragmentTransaction.commit();
  • 对于当FragmentA上启动FragmentB，返回是仍要返回到FragmentA时,需要加入到后退栈中，当你需要返回时仍保留FragmentA上的用户输入的信息，这时启动的时候需要用add(),如果用replace()的方式的话，其后退返回时，对应的之前后退栈中的Fragment数据会重新渲染，跟刚启动是一样，不会保留之前的用户操作,而针对add方式如果没设置背景的话需要hide隐藏，这是放返回时不用show，加入后退栈后返回其会自动show出来你对应hide的Fragment

    清空后退栈

    getFragmentManager().popBackStack()
  • 调用该方法相当于执行了一次back操作，自动退出栈。

  • 一般需要全部清空栈时，需要通过tag拿到对应的Fragment对象一个一个退，即

    Fragment home = fragmentManager.findFragmentByTag("Home");  
if(home!=null){  
//回退--退出后退栈栈  
home.getFragmentManager().popBackStack();
}  
Fragment talk = fragmentManager.findFragmentByTag("Talk");  
if(talk!=null){  
//回退--退出后退栈栈  
talk.getFragmentManager().popBackStack();  
}
